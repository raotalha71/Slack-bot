EXECUTIVE SUMMARY

Redstone Manufacturing operates two production facilities in Ohio with 320 staff across both sites. Equipment downtime is the company's primary operational cost driver. When a machine fails unexpectedly, the average unplanned stoppage costs Redstone approximately $18,000 per hour in lost output and labour. Over the past year the company experienced 14 unplanned stoppages averaging 3.2 hours each.

The current monitoring approach relies on operators manually logging machine status on paper sheets that are reconciled at the end of each shift. By the time a trend is visible in the data the failure has already occurred.

This proposal outlines a real-time production floor monitoring system that collects sensor data from critical equipment, identifies anomalies before they become failures, and alerts maintenance teams with enough lead time to schedule preventive intervention.

---

PROPOSED SOLUTION

Phase 1 — Sensor Integration and Data Pipeline (Months 1-3)
Install IoT sensors on the 24 highest-risk machines across both facilities. Build a data ingestion pipeline that streams sensor readings (temperature, vibration, pressure, cycle time) into a central time-series database every 30 seconds. A live dashboard displays current machine status across both sites.

Phase 2 — Anomaly Detection (Months 3-5)
Build a machine learning model trained on 18 months of historical sensor data (provided by Redstone) to identify patterns that precede failures. When anomaly thresholds are crossed, automated alerts are sent to the relevant maintenance technician via SMS and the operations dashboard.

Phase 3 — Maintenance Workflow Integration (Month 5-6)
Integrate alerts with Redstone's existing maintenance ticketing system (Fiix CMMS). When an anomaly is detected the system automatically creates a maintenance ticket with the machine ID, sensor readings, and recommended inspection checklist.
